## 单例模式的定义
单例模式（Singleton Pattern）是指确保一个类在任何情况下都绝对只有一个实例，并提供一个全局访问点。
隐藏其所有的构造方法。
属于创建型模式。

## 单例模式常见写法
1.饿汉式单例
2.懒汉式单例
3.注册式单例
4.ThreadLocal单例

## 单例模式的优点
在内存中只有一个实例，减少了内存开销
可以避免对资源的多重浪费
设置全局访问点，严格控制访问

## 单例模式的缺点
没有接口，扩展困难
如果要扩展单例对象，只有修改代码，没有其他途径

## 学习单例模式的只是重点总结
私有化构造器
保证线程安全
延迟加载
防止序列化和反序列化破坏单例
防御反射攻击单例
